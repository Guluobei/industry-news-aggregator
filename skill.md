# 行业新闻聚合推送器 Skill 定义

## 功能描述
根据用户配置的信息源（新闻网站、微信公众号等），自动收集、筛选、排序行业新闻，并推送到飞书、邮箱等渠道。支持问题识别与通知机制。

## 触发条件
- 用户请求："收集新闻"、"推送资讯"、"新闻简报"、"今日新闻"
- 定时任务：按cron表达式自动执行（默认每天9:00）

## 使用方式

### 1. 配置文件
用户编辑 `config.yaml`，填写三类信息：
- 信息源：网址或"公众号:名称"
- 筛选条件：行业、关键词、排除词、数量
- 推送渠道：飞书/邮箱/本地

### 2. 执行命令
```bash
python -m src.main --config config.yaml
```

### 3. 输出
- 飞书消息卡片（执行报告 + 新闻摘要）
- 飞书文档（完整新闻列表）
- 邮件HTML简报
- 本地Markdown/HTML/JSON文件

## 输入参数

| 参数 | 必填 | 说明 |
|------|------|------|
| --config | 是 | 配置文件路径 |
| --dry-run | 否 | 试运行，仅输出不推送 |
| --verbose | 否 | 详细日志 |

## 配置示例

```yaml
sources:
  - "https://finance.sina.com.cn"
  - "公众号:中国保险报"

filter:
  industry: "保险"
  keywords: ["保险", "医保", "监管"]
  exclude: ["娱乐", "八卦"]
  top_n: 10
  hours: 168

push:
  feishu:
    enabled: true
    doc_folder: "folder_token"
    notify_chat: "chat_id"
  local:
    enabled: true
    path: "./output"
```

## 问题识别机制

系统自动识别以下卡点并通知用户：
- 网址无法访问 / 页面404
- 公众号未找到 / 已注销 / 长期未更新
- RSSHub服务不可用
- 网站反爬拦截
- 筛选结果为空 / 不足
- 推送凭证错误 / 权限不足

每个问题包含：问题描述 + 可操作建议。

## 推送模板

三套固定美观模板，用户无需自定义：
1. **飞书消息卡片**：统计摘要 + 问题列表 + 新闻标题摘要 + 查看文档按钮
2. **飞书文档**：标题 + 统计 + 10条新闻（序号标题+来源时间+摘要+原文链接）
3. **邮件HTML**：渐变标题栏 + 统计卡片 + 新闻卡片列表 + 页脚
